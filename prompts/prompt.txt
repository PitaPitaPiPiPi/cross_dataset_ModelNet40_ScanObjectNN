## 1. ShapeNet 前処理を追加

新規作成：`scripts/build_shapenet.py`

目的：raw ShapeNet を既存 ModelNet と同じ形式へ変換する。

CLI：

```text
--shapenet_root
--out_root
--num_points 1024
--sample_surface_n 10000
--train_ratio 0.8
--workers
--seed 42
--skip_existing
--overwrite
```

要件：

* 入力形式：`.obj`, `.off`, `.ply`, `.npy` を自動判定。
* ShapeNetCore 形式 `synset_id/model_id/models/model_normalized.obj` に対応。
* 既に `class_name/train|test/*` がある場合は split を尊重。
* split がない場合は class ごとに deterministic train/test split を作り、`outputs/ShapeNet/split_manifest.json` に保存。
* `configs/shapenet_class_map.json` を追加し、canonical class name、directory aliases、optional synset id を持たせる。
* raw folder 名の揺れを class name / underscore / lowercase / aliases / synset id で吸収する。
* class dir は canonical 化する。例：`trash bin -> trash_bin`, `file cabinet -> file_cabinet`, `flowerpot -> flowerpot`。
* mesh は `trimesh.load(..., process=True)`、surface sampling、FPS、正規化の順で処理。
* point cloud は `.ply` を open3d 優先、失敗時 trimesh fallback、`.npy` は `(N,3)` / `(3,N)` を補正。
* 点数不足時は deterministic repeat/padding で `(num_points,3)` にし、warning を出す。
* 保存は既存 `save_npy_and_meta` を使う。
* metadata：`orig`, `dataset="ShapeNet"`, `class_name`, `class_dir`, `split`, `centroid`, `scale`, `num_points`, `source_format`。
* 終了時に class 別 train/test 件数と sample shape を表示。

## 2. CO3D 前処理を追加

新規作成：`scripts/build_co3d.py`

目的：CO3D v2 の各 sequence の `pointcloud.ply` を 1 sample として `.npy` 化する。

想定 raw 構造：

```text
CO3D_ROOT/<category>/<sequence_name>/pointcloud.ply
CO3D_ROOT/<category>/set_lists/*.json
CO3D_ROOT/<category>/{frame_annotations.jgz,sequence_annotations.jgz}
```

CLI：

```text
--co3d_root
--out_root
--categories apple,backpack,...
--num_points 1024
--train_ratio 0.8
--workers
--seed 42
--use_set_lists
--skip_existing
--overwrite
--strict
```

CO3D 50 categories：

```text
apple, backpack, ball, banana, baseballbat, baseballglove, bench, bicycle, book, bottle,
bowl, broccoli, cake, car, carrot, cellphone, chair, couch, cup, donut,
frisbee, hairdryer, handbag, hotdog, hydrant, keyboard, kite, laptop, microwave, motorcycle,
mouse, orange, parkingmeter, pizza, plant, remote, sandwich, skateboard, stopsign, suitcase,
teddybear, toaster, toilet, toybus, toyplane, toytruck, tv, umbrella, vase, wineglass
```

要件：

* `configs/co3d_class_map.json` を追加し、上記 50 categories を定義。
* `--categories` 未指定時は config 全カテゴリを対象にする。
* category 単位・逐次処理を最重要視する。既存 `outputs/CO3D/<category>` は削除しない。
* split は sequence 単位。train/test に同一 sequence を混在させない。
* 既定は deterministic sequence split。`outputs/CO3D/split_manifest.json` に保存。
* `--use_set_lists` 指定時のみ `set_lists` 使用を試みる。
* `set_lists` が frame-level split で同一 sequence を train/test/val に跨らせる場合は使用せず deterministic split に fallback。
* val は conflict がなければ test に merge。
* `.ply` は open3d 優先、失敗時 trimesh fallback。
* 点数不足時は deterministic repeat/padding で `(num_points,3)` にし、warning。
* 保存は既存 `save_npy_and_meta`。
* metadata：`orig`, `dataset="CO3D"`, `category`, `sequence_name`, `split`, `centroid`, `scale`, `num_points`, `source_format="ply"`。
* `--strict` なしでは存在するカテゴリだけ処理可能。`--strict` ありでは不足カテゴリを error。
* 終了時に category 別 train/test 件数と sample shape を表示。

## 3. CO3D category-wise download + preprocess を追加

新規作成：`scripts/download_and_build_co3d_category.py`

目的：CO3D 全体は巨大なため、category ごとに download → preprocess → `outputs/CO3D` へ追記する。

CLI：

```text
--co3d_repo external/co3d
--download_folder raw_datasets/co3d
--category apple
--categories apple,backpack,...
--out_root outputs
--num_points 1024
--workers
--seed 42
--download_only
--process_only
--clear_archives_after_unpacking
--overwrite
```

要件：

* CO3D repo は以下のように外部 clone されている前提：

```bash
mkdir -p external
git clone https://github.com/facebookresearch/co3d.git external/co3d
```

* 内部で公式 downloader を subprocess 実行する。

```bash
python external/co3d/co3d/download_dataset.py \
  --download_folder raw_datasets/co3d \
  --download_categories apple \
  --clear_archives_after_unpacking
```

* `--download_only` でない場合、対象 category のみ `build_co3d.py` 相当で前処理する。
* `--process_only` では download せず前処理のみ。
* ダウンロード済み category は再 download を避ける。
* 失敗時は category 名を明示して error。
* `raw_datasets/co3d/_in_progress` が残る場合の注意を README に記載。

## 6. 前処理済み ShapeNet / CO3D / S2C を個別実行可能にする

既存の ModelNet40 / ScanObjectNN / M2S と同じ抽象度で、以下を別々に読み込み・実行可能にしてください。

```text
modelnet        -> outputs/ModelNet
scanobjectnn    -> outputs/ScanObjectNN
m2s             -> outputs/modelnet_scanobjectnn
shapenet        -> outputs/ShapeNet
co3d            -> outputs/CO3D
s2c             -> outputs/shapenet_co3d
```

受け付ける alias：

```text
ShapeNet, shapenet
CO3D, co3d
shapenet_co3d, s2c, S2C
```

要件：

* 既存 dataset loader / runner がある場合、それを拡張する。
* 可能なら single dataset と cross dataset の loader を共通化する。
* 破壊的変更になりそうなら薄い wrapper を追加する。
* single dataset、つまり ShapeNet / CO3D は class-name based directory を読み、label は deterministic class order から作る。
* ShapeNet の class order は `configs/shapenet_class_map.json` または S2C map の ShapeNet 部分に従う。
* CO3D の class order は `configs/co3d_class_map.json` に従う。
* CO3D は逐次処理されるため、`--strict` なしでは存在 category のみで実行可能にする。
* S2C は class-id based cross dataset として読み、session 情報は `configs/shapenet_co3d_sessions.json`、class map は `configs/shapenet_co3d_class_map.json` を使う。
* S2C の label は原則 global class_id 0〜88。既存 M2S が local label 設計なら既存に合わせつつ、global class_id を metadata に保持する。
* S2C も `--strict` なしでは存在 class_id のみで実行可能、`--strict` ありでは 0〜88 の不足を error。
* 既存分岐がある場合は `shapenet`, `co3d`, `s2c` を追加する。

推奨追加：

```text
scripts/smoke_test_dataset.py
```

CLI：

```text
--dataset {modelnet,scanobjectnn,m2s,shapenet,co3d,s2c}
--data_root
--split train|test
--session
--strict
--num_points 1024
```

smoke test は前処理済みデータを読むだけにし、以下を表示する：

```text
dataset, split, session if any, number of classes, number of samples, first sample shape, first label, first metadata if any
```

shape が `(1024,3)` でなければ error。
